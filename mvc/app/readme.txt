Как это всё работает

1. Точкой входа является app.js, который находит на странице элементы помеченные
   аттрибутом data-controller, берет значение(name) из этого аттрибута и ищет
   в папке ./controllers файл с именем name + 'Controller';

   Файлы из папки получаются с помощью require.context и если находит такой файл,
   то создает новый контроллер new controller(el); и передает ему его корневой элемент

2. BaseController имеет единственный метод renderView, который получает на вход
   имя файла в котором хранится шаблон(viewName) и данные для этого шаблона
   (viewModel). Создает новый шаблон с помощью шаблонизатора(lodash) и отдает
   ему данные. Полученный html записывает в innerHTML корневого элемента

3. ProfileController наследует BaseController имеет метод index,
    который вызывает базовый метод renderView отдает ему имя 
    представления('profile') и данные(viewModel) полученные из файла
    ./model/profileModel.js.

    Единственной логикой которую выполняет контроллер Profile является
    склеивание firstName и lastName в fullName


Как это всё запустить

    0. Установить node.js

    1. Открыть консоль или powershell(лучше под админом)
    2. Перейти в папку app(перейти в консоли, а не в командере или файлэксплорере, чтобы в консоли был правильный путь mvc/app>)
    3. Выполнить команду 'npm install' (без кавычек :)), у вас должна появиться папка node_modules
    4. Выполнить команду 'npm run build', в корневой папке рядом с index.html и style.css должен появиться файл main.js
    5. Запустить index.html
    6. После внесения любых изменений в папке app(модель, контроллер, представление) нужно выполнять 'npm run build' чтобы пересобрать файл main.js
